LinearAlgebra lec02

lec02这节课是图形学需要用到的线代知识的复习，比较简单，这里只简要列出，主要是要知道在图形学的应用。
vector：向量的加法、点乘dot product、叉乘cross product。点乘得到一个数值，叉乘得到一个新的向量。
matrix：矩阵加法、矩阵和向量相乘、矩阵和矩阵相乘、转置、单位矩阵、逆矩阵、伴随矩阵

点乘：${
{ \overrightarrow{a} \cdot \overrightarrow{b} }
= { {\lVert \overrightarrow{ a } \rVert} {\lVert \overrightarrow{ b } \rVert} {\cos\theta} }
}$ 点乘性质：交换律、结合律、分配律

叉乘：${
{ \lVert \overrightarrow{a} \times \overrightarrow{b} \rVert }
= { \lVert \overrightarrow{a} \rVert }{ \lVert \overrightarrow{b} \rVert }\sin\theta
}$ ；右手螺旋判断方向。叉乘结果的模在大小上等于两个向量组成的三角形面积

矩阵性质：结合律、分配律、没有交换律。

  • $ (AB)^T = {B^T}{A^T} $
  • $AA^{-1}= A^{-1}A=I$
  • $(AB)^{-1}=B^{-1}A^{-1}$

在图形学上的应用：

  • 点乘
    • 求任意两个向量之间的夹角（例如光源在表面之间角度的余弦）。
    • 求一个向量在另一个向量上的投影
    • 根据结果正负判断一个向量对于另一个向量是朝前还是朝后的。
  • 叉乘
    • 判断向量在另一个向量的左边还是右边 ； ${
      { \overrightarrow{a} \times \overrightarrow{b} >0 }
      }$ 表示b在a左边
    • 判断点P在三角形内还是外（三次按顺序的叉乘方向相同的话在内）
  • 矩阵
    • 常用矩阵来表示变换信息：平移、旋转、缩放、切变等 Translation, rotation, shear, scale

Transformation lec03+lec04部分

先具体讲几种2D的变换：Scale Transform 缩放1变换矩阵、Reflection Matrix对称2变换矩阵、Shear Matrix切变3变换矩阵、Rotation Matrix旋转4变换矩阵。我们发现以上四种都可以用一个矩阵乘以输入可以得到输出的坐标，这种叫做线性变换。平移也是常用的变换，但不是线性变换。为了不特殊处理平移的情况，引入齐次坐标Homogeneous coordinates来统一处理各种变换，具体做法是增加一个维度。
然后还有6仿射变换Affine和7逆变换。仿射= 线性变换+平移
然后3D是在lec04的，也放在这里，与2D类似，不具体讲。

0 why study transformation

  • Modeling模型变换: translation/rotation/scaling
  • Viewing视图变换: (3D to 2D) projection

1、2D transformations

  1. 缩放
  2. 对称
  3. 切变：竖直方向是没有变的/y不变，沿水平拉伸。
  4. 旋转矩阵的逆就是它的转置：正交矩阵 $A^{-1} = A^T$

    主要是找到变换前后对应的关系就能表示出来。
    这几种都是线性变换： 输出坐标 = 矩阵 乘以 输入坐标

  5. 平移

齐次坐标

why？平移不能用线性变换表示，但是我们又不想把它作为特殊情况处理。所以引入齐次坐标，把所有的变换写成一个矩阵乘以一个向量。
具体做法是增加一个维度。

  1. 仿射变换Affine = 线性变换+ 平移 （先线性再平移）
  2. 逆变换

那么这些变换我们可以组合、分解。
通过组合可以表示复杂的变换。反过来也可把复杂的变换分解。要注意顺序（因为做变换其实就是左乘一个矩阵，矩阵的乘法是不满足交换律的）

2、 3D transformations

与2D类似，也可以用齐次坐标。

  • 绕xyz轴旋转
  • 可以把3D的旋转分解成xyz轴 飞机的例子
    • $R_{xyz}(\alpha, \beta, \gamma) = R_x(\alpha)R_y(\beta)R_z(\gamma)$
  • Rodrigues公式：得到围绕一个旋转轴旋转任意角度后的向量

Blinn-Phong Reflection Model: 三种光照的组合

$$\begin{split}
L &= L_a + L_d + L_s \
&= k_aI_a + k_d(I/r^2)max(0, \mathbf{n} \cdot \mathbf{l}) + k_s(I/r^2){max(0, \mathbf{n \cdot h})}^p
\end{split}$$

  • 环境光Ambient lighting $L_a = k_aI_a$ 给模型所有位置均匀的光照，常数。
  • 漫反射光Diffuse Reflection，光线打到物体的表面会被均匀的散射到各个方向
    • $L_d = k_d(I/r^2)max(0, \mathbf{n} \cdot\mathbf{l})$
    • 光线照射到模型的方向与模型法线，夹角越小的地方，光照越强
  • 镜面反射光/高光Specular highlights，观察方向和光线的镜面反射方向接近的地方，模型会产生高光
    • $$\begin{split}
      L_s &= k_s(I/r^2){max(0, \cos\alpha)}^p \
      &= k_s(I/r^2){max(0, \mathbf{n \cdot h})}^p
      \end{split}$$

几个k分别是对应的系数

实时渲染管线：3D场景到2D图的过程

  • Vertex Processing
    主要是把我们输入的顶点进行MVP变换，得到屏幕空间的顶点表示，方便后续处理。

  • Triangle Processing
    按照一定顺序，将顶点组成不同的三角形

  • Rasterization
    对每个像素进行采样，判断像素在不在三角形内

  • Fragment Processing -> Z-buffer 、 Shading、Texture mapping
    判定每个像素位置哪些物体可见，使用Z-Buffer做深度测试
    计算shading point（根据不同的着色频率决定不同粒度的shading point）的颜色，并着色
    这个阶段也会做纹理映射，简单说就是把贴图（颜色、法线等等）给映射到物体的不同位置上，后面会详细说明这一部分

  • Framebuffer Operations
    ​ 根据像素大小，最终生成一个用于显示的Framebuffer

Introduction to Computer Graphics

What is Computer Graphics?

The use of computers to synthesize and manipulate visual information.
使用计算机来合成和操作视觉信息。

Why Study Computer Graphics

applications

  • Video Games（写实、风格化渲染，看画面亮不亮可以体现全局光照效果）
  • Movies（特效是图形学比较简单的应用，一些仿真效果往往很难实现，阿凡达的面部捕捉具有里程碑意义）
  • Animations（疯狂动物城中对毛发的模拟）
  • Design（CAD、在虚拟环境中预览视觉效果：汽车、家居）
  • Visualization（科学、工程、医学、新闻图表）
  • Virtual Reality
  • Augmented Reality
  • Digital Illustration 数字图像处理 (Photoshop)
  • Simulation 模拟（精确的物理模拟仿真：尘土、黑洞对光线的影响）
  • Graphical User Interfaces
  • Typography 排版/字体（运用了点阵和矢量，The Quick Brown Fox Jumps Over The Lazy Dog用尽了26个英文字母经常被拿来测试字体）

Fundamental Intellectual Challenges

  • Creates and interacts with realistic virtual world 创建一个真实的虚拟世界并与之交互
  • Requires understanding of all aspects of physical world 理解物理世界的各个方面（运动，光照等等）
  • New computing methods, displays, technologies 新的计算方法、图形显示技术

Technical Challenges

  • Math of (perspective) projections, curves, surfaces(透视)投影、曲线、曲面的数学知识
  • Physics of lighting and shading 光线和阴影的物理学
  • Representing / operating shapes in 3D 在三维中的表现/操作形状
  • Animation / simulation 动画/模拟
  • 3D graphics software programming and hardware 不包括硬件编程教学
    重点强调：计算机图形学不是GUI，不是学习一种图形学接口就叫做学会了计算机图形学

Computer Graphics is AWESOME！

Course Topics

  • Rasterization 光栅化

    • 将几何图元（3D 三角形/多边形）投影到屏幕上
      Project geometry primitives (3D triangles/polygons) onto screen
    • 将投影图元分解为片段（像素）
      Break projected primitives into fragments (pixels)
    • 视频游戏（实时应用）的黄金标准
      Gold standards in Video Games (Real-time Applications)

  • Curves and Meshes 曲线和网格

    • 如何在计算机图形学中表示几何图形（曲面、表面细分时保持拓扑结构）
      How to represent geometry in Computer Graphics

      如何表示光滑曲线;如何表示曲面;
      如何通过通过细分将简单的曲面变成更复杂的曲面;
      在形状发生变化时如何保持拓扑结构

  • Ray Tracing 光线追踪

    • 从相机出发向每个像素发射射线（计算交点和阴影；继续反射光线直到击中光源）
      shoot rays from the camera through each pixel(Calculate intersection and shading; Continue to bounce the rays till they hit light sources)
    • 动画/电影的黄金标准（离线应用）目前已经有了实时光线追踪的算法
      Gold standard in Animations/Movies (Offline Applaction)

      慢，但能生成很真实、质量更高的画面（动画电影中常用）
      图形学中有很多事情我们管它叫Trade off（代价）（为了达成某个目标不得不牺牲一些其他的目标）
      但现在还是出现了两全其美的解决方法：实时光线追踪

  • Animation / Simulation 动画/模拟

    • 关键帧动画 Key frame Animation
    • 质量弹簧系统 Mass-spring System

      弹性球落地，如何挤压，弹跳 ; 布料模拟落下等等

Computer Graphics VS Computer Vision

计算机图形学是从三维模型生成图像（渲染）或三维空间中的模拟（模拟）；
计算机视觉是通过图像语义分割进行建模或依赖深度学习由图像生成其他图像

一般来讲图形学研究的是将Model变成Image的过程，反之则是计算机视觉。
另外，Model的表示与模拟也是计算机图形学研究的方向，涉及到模型的简化、细分，材质表示等。
图形学不是计算机视觉。 计算机视觉：猜测、预测、分析处理
计算机视觉是理解这个世界，计算机图形学是创造这个世界
